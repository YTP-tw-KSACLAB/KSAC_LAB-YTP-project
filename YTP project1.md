# 行旅台北 ── 行腳台北 × 行樂台北

## 專案名稱：Taipei Vibe (台北漫遊) - 風格客製化一日旅遊計畫

### 1. 解決方案與核心功能 (Solution)
* **AI 旅遊顧問（對話介面）：** 透過自然語言輸入需求，AI 自動提煉出用戶的「旅遊風格標籤」（如：歷史、美食、夜生活、無障礙）。
* **Agentic Automation：** 透過AI agents 自動規劃與協助訂房、訂飯店與買票，大量節省時間。
* **動態時間軸生成：** 結合交通 API，自動生成包含「景點 A $\rightarrow$ 步行 5 分鐘 $\rightarrow$ 捷運紅線 $\rightarrow$ 停留 2 小時」的精確一日遊時間表。
* **突發狀況應變：** 若遇到下雨或行程延誤、道路事故等，可一鍵點擊「重新規劃」，VLM 會立即尋找附近的室內備案。一鍵重新規劃 (Instant Re-route)： * 若氣象預報顯示降雨機率 >70%，或當前景點出現大規模延誤，AI 主動推送「室內備案」供用戶一鍵替換。
* **深度個性化：** 社交媒體與數位足跡整合 **社群數據探勘 (Social Media Personalization)：** 透過 API 整合用戶在 Instagram、Pinterest 或小紅書上的收藏夾與按讚紀錄，自動識別視覺偏好（如：文青工業風、大自然壯麗感）。
*  **合法旅宿資料比對** 串接臺灣旅宿網合法名錄，透過地址解析與模糊比對，判斷訂房平台上的住宿是否已合法登記
*  **住宿 URL 智慧解析** 使用者貼入訂房連結，系統自動擷取住宿資訊（名稱、地址、圖片），進行合法性比對。
*  **風險預警通知:** 比對結果若存疑，系統主動推送風險提示，說明風險類型（消防/衛生/法規），並推薦 3 家鄰近合法替代旅宿。
*  **多語系 AI 介面:**支援英、日、韓、東南亞語系自動切換，AI Agent 對話以使用者語言回應，服務國際旅客。
*  **合法旅宿地圖瀏覽:**整合地圖介面，顯示周邊合法旅宿分布、評分與即時房況，支援篩選條件與收藏功能。
*  **黑名單通報機制:**使用者可通報疑似非法住宿，系統彙整後協助相關單位進行稽查追蹤。
*  **分析用戶過去發布的旅遊貼文，自動標記「高頻標籤」（例：重度咖啡愛好者、古著獵人）。**

數位足跡同步： 連結 Google Calendar 或電子郵件，自動偵測已預訂的機票與飯店資訊，將其設為行程的「硬性錨點」。


**全域數據驅動：多源動態評論與排名系統** 。打破單一平台偏見，利用全網數據確保景點與餐廳的品質穩定。

全網評價權重 (Multi-Source Ranking)： * 自動抓取 Google Maps、TripAdvisor、Yelp 以及在地論壇（如：PTT、Reddit）的評論 也使用Brave API。

情緒分析 (Sentiment Analysis)： AI 自動過濾掉過時的資訊（如：半年前的裝潢汙點）或疑似水軍的評論，優先採納近一個月內的真實反饋。

* **社群輿情探勘引擎: 透過 NLP 識別熱門話題與新興景點**。

*  **即時活動推播通知:** 整合台北活動開放資料，主動推播使用者附近 48 小時內的展演、市集與限時優惠資訊。
*  **行程分享與協作:** 使用者可將生成行程分享給同伴，支援多人協作編輯與即時同步。並支援透過3D VR map 等方式提供遠端體驗功能，減少預期落差
*  **即時同步掌握人流資料** 透過大量用戶資料，改進行程安排避免人群密度高，且用戶可及時反饋





### 2. 技術架構與推薦 API 工具 (Tech Stack & APIs)

#### 核心技術
* **LLM (大語言模型)：** 負責理解用戶意圖與最終行程排版生成。
* **RAG (檢索增強生成)：** 建立台北景點知識庫。先從知識庫檢索符合用戶風格的景點，再交由 LLM 排序。這樣能避免 AI 產生「幻覺」（推薦不存在的店家）。

#### 推薦 API 與工具清單
* **LLM 引擎：**
    * **Google Gemini API** 或 **OpenAI API (GPT-4o)**：處理對話與排程邏輯。
* **RAG 與資料庫：**
    * **LangChain / LlamaIndex：** 串接 LLM 與資料庫的必備框架。
    * **Vector DB (向量資料庫)：** **Pinecone**、**ChromaDB** 或 **Qdrant**，用於儲存景點描述的向量，實現語意搜尋（例如搜「安靜看書的地方」，能精準對應到圖書館或靜謐咖啡廳）。
* **地理資訊與交通 API (極度重要)：**
    * **交通部 TDX (運輸資料流通服務平臺)：** **必接！** 包含全台公車、捷運、YouBike 2.0 即時站點資訊與路線規劃。
    * **Google Maps Platform：**
        * *Places API：* 獲取景點的營業時間、評分、照片。
        * *Directions API / Distance Matrix API：* 計算點到點的精確交通時間。
    * **臺北市資料大平臺 (data.taipei)：** 抓取台北市特色活動、市集、藝文展覽等即時 Open Data，讓行程具備時效性。
    * **台灣旅宿網：** 串接臺灣旅宿網合法名錄，透過地址解析與模糊比對，判斷訂房平台上的住宿是否已合法登記。

擴充 API 與工具清單
A. 社交媒體與個性化數據 (Social & Personalization)
Instagram Graph API： 在用戶授權下，抓取用戶的收藏夾（Media Insights）與標籤熱點，分析其視覺偏好。

Google Calendar API： 自動讀取用戶已訂購的機票/餐廳行程，作為行程生成的硬性邊界。
全網數據與評論權重 (Web Intelligence)
SerpApi / ValueSerp： 抓取 Google 搜尋結果中的即時評論摘要與在地論壇討論，規避單一平台的資訊偏差。

TripAdvisor API： 獲取全球通用的景點權重排名與景點分類標籤。

Yelp Fusion API： 針對美食與夜生活提供高密度的深度評論數據（適用於國際旅客）。

Brave Search API / DuckDuckGo API 通用搜尋

### 3. 使用者介面設計 (Solution) 以安全環保旅遊為中心的旅遊社交App : SnapTravel

這是一個結合了 **Instagram 的社群感**、**Google Maps 的導航力**與 **AI Agent 代辦能力** 的全新旅遊 App 提案。以下是針對 `SnapTravel` 所設計的使用者介面（UI）與互動流程架構：

---

### 3. 使用者介面設計 (User Interface Design) — SnapTravel

`SnapTravel` 的設計核心在於「**從靈感探索到行程落地的一站式社群體驗**」。介面風格採用簡潔、高對比的「微光設計」，強調地圖與影像的結合。

#### 3.1 核心五大功能板塊 (Tab Bar)

1.  **Vibe 探索 (靈感流)**：Instagram 化的旅遊貼文牆。
2.  **Smart Map (智慧地圖)**：整合即時人流、導航與相似路徑旅伴。
3.  **AI 行程大腦 (Agentic Core)**：對話式規劃、一鍵重新規劃與代訂服務。
4.  **Squad 揪團與聊聊 (Social)**：多人協作與即時溝通。
5.  **旅程檔案 (My Journey)**：個人數位足跡與合法旅宿認證。

---

#### 3.2 詳細頁面設計與功能

##### A. Vibe 探索牆：從「看」到「行」
* **視覺化卡片**：每張照片/短影音右下角都有一個「**Vibe Clone**」按鈕，點擊後 AI 自動提取該貼文的地點、風格，並詢問是否加入你的計畫。
* **風格標籤導航**：頂部有 `#文青咖啡`、`#台北夜景`、`#綠色旅行` 等滑動式標籤，點擊後會自動過濾地圖與行程推薦。
* **相似路徑推薦**：當你正在瀏覽某人的大稻埕行程時，系統會顯示：「有 5 位朋友也在看這個區域」，點擊可進入「臨時小隊」。

##### B. AI 行程大腦：對話式 Agent
* **智慧卡片 UI**：AI 不只回覆文字，會直接推播「**可操作卡片**」。
    * *例：*「我看中這家餐廳，要訂嗎？」卡片顯示：[餐廳名 | 剩餘位子 | 綠色認證]，下方按鈕：[**預訂 19:00**] [**加入行程**]。
* **一鍵重新規劃 (Magic Re-route)**：遇雨時，主畫面會浮現「**雨天警報**」小部件，點擊後，原本的戶外行程卡片會「翻轉」變成鄰近的室內美術館或咖啡廳。
* **Agentic 訂房代辦**：輸入連結（如 Booking.com），AI 會在對話框彈出：「**合法性驗證中...** ✅ 合法旅館。目前與官網比價便宜 200 元，需要幫你代訂嗎？」

##### C. 智慧地圖與導航：即時互動層
* **3D VR 預視層**：在地圖點擊景點，可彈出 3D 環景或 VR 視窗，減少實體預期落差。
* **人流熱圖 (Live Density)**：地圖上以顏色顯示人群密度，AI 會提醒：「目前西門町人潮擁擠，建議先去旁邊的剝皮寮，步行僅 10 分鐘」。
* **路徑旅伴 (Route Match)**：地圖上會顯示淡淡的藍色光點，代表有其他 SnapTravel 用戶有相似行程。
    * *功能*：點擊光點可發送「**Hi! 要一起分擔 Uber 嗎？**」或「**這家店要排很久嗎？**」的即時訊息。

##### D. Squad 群組協作：旅遊版 Slack + IG Chat
* **共同編輯模式**：群組頂部懸浮「**共同時間軸**」，任何人拖拽景點，群組成員會即時看到時間表變動。
* **分帳系統 (Split-Bill)**：整合在聊天室中，Agent 會自動偵測訂位資訊，提醒成員分擔費用。
* **多語系翻譯**：若為國際團，聊天室支援即時雙向翻譯，對話泡泡可一鍵轉換成各國語言。

#### 3.3 關鍵互動設計 (Key Interaction)

| 情境 | 使用者動作 | 系統/Agent 反應 |
| :--- | :--- | :--- |
| **發現靈感** | 在 IG 風格牆看到一張台北 101 的美照 | 點擊 [Clone]，AI 自動安排最佳拍攝位點並插入當前行程。 |
| **住宿驗證** | 貼上 AirBnB 連結到聊天室 | Agent 立即回傳：⚠️ 非合法登記。並附上 3 家鄰近合法飯店對比圖。 |
| **交通移動** | 抵達公車站點 | 介面自動切換為「TDX 即時公車卡片」，顯示：🚌 307 路還有 3 分鐘。 |
| **行程延誤** | 在某景點停留過久 | AI 推送：「看來你在這裡玩得很開心！後續行程已自動向後順延 30 分鐘，並已幫你修改餐廳訂位。」 |

#### 3.4 視覺語言 (Visual Identity)
* **配色方案**：大稻埕紅 (磚紅) + 捷運藍 (商務質感) + 永續綠 (環保標章)。
* **字體**：採用易讀的 Sans-serif，數字部分採用等寬字體，方便閱讀交通時間表。
* **圖示**：使用大量動態 Lottie icon（如：下雨時 icon 會滴水，提醒重新規劃）。

這樣的設計能確保 `SnapTravel` 不只是一個冷冰冰的規劃工具，而是一個能與朋友互動、能讓 AI 幫忙跑腿、且能隨時應變意外的「數位旅遊嚮導」。
這部分非常重要，將 `SnapTravel` 從一個純工具轉向具備自我獲利能力的商業生態系。以下是針對「商業可行性與獲利模式」的詳細規劃：

### 4. 商業可行性與獲利模式 (Business Viability)

`SnapTravel` 的獲利策略結合了 **B2B 合作推廣**、**B2C 增值服務** 與 **數據生態循環**，確保平台在提供價值的同時具備持續經營的動能。

#### 4.1 B2B 深度合作：精準行銷與分潤 (Affiliate & Partnerships)
* **合法旅宿推薦佣金**：
    * 與通過驗證的「合法旅宿」深度對接。當用戶透過 AI Agent 完成訂房時，系統抽取 5-10% 的引流佣金。
    * **獨家優惠券 (In-app Coupons)**：AI 根據用戶的風格標籤（如：咖啡愛好者），在行程規劃中主動推播鄰近合作店家的折扣券，提高轉化率。
* **店家精準導流 (Sponsored Pins)**：
    * 合作餐廳或景點可付費在「智慧地圖」中獲得優先曝光（標註為「SnapTravel 推薦」），但 AI 仍會誠實標註其與用戶風格的匹配度。

#### 4.2 點數經濟與用戶黏著 (SnapPoints Ecosystem)
* **點數賺取 (Earn)**：
    * **綠色旅遊獎勵**：使用 YouBike、搭乘捷運或入住環保標章旅館，系統自動發放 `SnapPoints`。
    * **貢獻價值**：用戶通報非法住宿、撰寫深度評論、或分享高人氣行程被他人 [Clone] 時，可獲得點數獎勵。
* **點數交易與兌換 (Trade)**：
    * 點數可直接抵扣合作店家的消費、兌換台北捷運一日票，或在 App 內購買虛擬禮物贈送給行程分享者。

#### 4.3 訂閱制服務 (Premium Subscription: SnapTravel Pro)
針對深度旅遊者與數位遊牧民族提供增值服務：
* **無限 AI 規劃**：免費用戶每日生成行程次數有限，訂閱用戶可享有無限次 Agentic 規劃與「一鍵重新規劃」。
* **AI 行程經紀人**：解鎖「主動預約」功能。訂閱用戶只需口頭交辦，AI 直接完成餐廳訂位、票券購買與住宿結帳。
* **獨家 3D/VR 體驗**：解鎖特定熱門景點的高清 3D 遠端預視功能，協助做出更精確的旅行決策。
* **免廣告社群體驗**：隱藏贊助商內容，提供純淨的 Vibe 探索牆。

#### 4.4 數據資產與政府協作 (B2G/Data Insights)
* **人流預測大數據**：
    * 彙整匿名化的用戶旅遊意圖數據，提供給臺北市政府觀傳局或交通局，作為城市規劃、人流疏散、節慶活動設計的數據參考。
* **安全認證服務**：
    * 透過大量的合法旅宿比對數據，協助相關單位即時更新「非法住宿黑名單」，降低觀光風險。
